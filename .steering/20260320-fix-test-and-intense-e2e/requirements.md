# 要求内容

## 概要

テスト失敗1件の修正と、重厚長大・複雑システムを模した過激なE2Eテストの追加。

## 背景

- `test_nonexistent_file_cached_as_none` が javalang 未インストール環境で失敗している（早期 return でキャッシュされない設計なのに skip 分岐がない）
- 現状のテストは単純なケースのみ。実際の大規模 Java プロジェクト（マルチモジュール、定数クラス、継承、getter、複合参照）での動作保証がない

## 実装対象の機能

### 1. 失敗テストの修正
- `test_nonexistent_file_cached_as_none` に `_JAVALANG_AVAILABLE` が False の場合は skip する分岐を追加
- javalang 未インストール環境での全テストがパスすること

### 2. 重厚長大 E2E テスト用フィクスチャ
- マルチモジュール構成の大規模 Java プロジェクトを模したフィクスチャ一式
  - 定数クラス（複数、階層的）
  - エンティティ（フィールド＋getter＋非標準getter）
  - サービス層（定数・フィールド・getter を複雑に参照）
  - ローカル変数経由の参照
  - アノテーション、継承、インターフェース
- 複数キーワードの grep ファイル（50〜100行規模）
- E2E テストクラス（パイプライン全体を通す統合テスト）

## 受け入れ条件

### テスト修正
- [ ] `python -m unittest test_analyze` で failures=0
- [ ] javalang 未インストール時に skip 数が変わらないこと（+1 で計 11 件）

### 過激 E2E テスト
- [ ] 複数の直接参照（定数・変数・条件・return・引数・アノテーション）が TSV に出力される
- [ ] 定数（project スコープ）の間接参照が複数ファイルにわたって検出される
- [ ] フィールド（class スコープ）の間接参照が同一クラス内で検出される
- [ ] getter 経由参照が検出される（find_getter_names で複数候補）
- [ ] ローカル変数（method スコープ）の間接参照がメソッド内で検出される
- [ ] バイナリ行・空行が含まれても正常終了する
- [ ] TSV ソート順（文言→ファイルパス→行番号）が正しい
- [ ] 処理統計が正確に集計される

## スコープ外

- javalang のインストール（テスト環境には未インストール、フォールバック動作を検証）
- 実際の Shift-JIS ファイル読み込み（フィクスチャは UTF-8 で記述し errors="replace" で対応）
