# 要求内容

## 概要

Pro*C（埋め込みSQL付きCコード）のgrep結果を自動分類・使用箇所洗い出しするツールを新規作成する。
Javaツール（`analyze.py`）と同等の機能を、Pro*C言語の特性に合わせて実装する。

## 背景

既存の `analyze.py` はJava専用。大規模Pro*Cプロジェクトでもgrepヒットが数万行規模になることがあり、
Pro*C固有の構文（EXEC SQL文・ホスト変数・`#define`定数など）を考慮した自動分類が必要。

Javaとの主な違い:
- AST解析ライブラリが事実上存在しない → **正規表現主体**（javalangフォールバックと同じ立場が本線）
- getterのような命名規則がない → getter追跡ステージ（第3段階）は対象外（MVP時点）
- 参照の間接経路は `#define` 定数 と ホスト変数（`:varname`）が主役
- 対象ファイル拡張子: `.pc`（Pro*Cソース）, `.h`（ヘッダ）

## 実装対象の機能

### 1. grep結果パース（Javaツールと共通概念）

- `input/*.grep` を読み込み、`filepath:lineno:code` 形式の行をパース
- バイナリ通知行・空行・不正フォーマット行はスキップ

### 2. Pro*C向け使用タイプ分類（第1段階）

正規表現ベースで以下の7種に分類する:

| 優先度 | 使用タイプ | 検出パターン |
|--------|----------|------------|
| 1 | EXEC SQL文 | `\bEXEC\s+SQL\b` |
| 2 | #define定数定義 | `^\s*#\s*define\b` |
| 3 | 変数代入 | `\w[\w\s\*\[\]]*\s*=\s*` (宣言 or 代入) |
| 4 | 条件判定 | `\bif\s*\(|\bwhile\s*\(|\bstrcmp\s*\(|[!=]=` |
| 5 | ホスト変数宣言 | EXEC SQL BEGIN/END DECLARE SECTIONの文脈 |
| 6 | 関数引数 | `\w+\s*\(` |
| 7 | その他 | 上記非マッチ |

### 3. 間接参照追跡（第2段階）

| 直接参照の種類 | 追跡スコープ |
|---|---|
| #define定数定義 | プロジェクト全体（`.pc`, `.h` ファイル） |
| ローカル変数代入 | 同一関数内（`{` `}` の対応で範囲特定） |
| ホスト変数宣言 | 同一ファイル内 |

### 4. TSV出力・処理レポート

- Javaツールと同一フォーマット（UTF-8 BOM付き）
- 列構成: 文言・参照種別・使用タイプ・ファイルパス・行番号・コード行・参照元変数名・参照元ファイル・参照元行番号

## 受け入れ条件

### grep結果パース
- [ ] `input/` の `.grep` ファイルを全件自動検出する
- [ ] `filepath:lineno:code` 形式をパースする（Windowsパス対応）
- [ ] バイナリ通知行・空行・不正行をスキップしカウントする
- [ ] `input/` が空ならexit code 1でエラー出力する

### 使用タイプ分類
- [ ] 7種の使用タイプに正規表現で分類する
- [ ] EXEC SQL文を最優先で検出する（アノテーション相当の特殊構文）
- [ ] 分類不能なものは「その他」として出力する（スキップしない）

### 間接参照追跡
- [ ] `#define NAME "keyword"` の定義行からNAMEを抽出し、プロジェクト全体を追跡する
- [ ] ローカル変数代入は同一関数スコープ内のみを追跡する
- [ ] ホスト変数宣言は同一ファイル内を追跡する
- [ ] 参照元変数名・参照元ファイル・参照元行番号を記録する

### TSV出力
- [ ] Javaツールと同一の列構成・エンコーディング（UTF-8 BOM付き）で出力する
- [ ] ソート順: 文言 → ファイルパス → 行番号（昇順、行番号は数値ソート）

### CLI
- [ ] `--source-dir`（必須）, `--input-dir`（デフォルト `input/`）, `--output-dir`（デフォルト `output/`）をサポートする
- [ ] `run_proc.sh` / `run_proc.bat` でワンライン実行できる

## 成功指標

- サンプルPro*Cファイル群を使ったE2Eテストで、期待TSVと全行一致すること
- 直接参照・#define間接参照・ホスト変数間接参照の全パターンが出力されること

## スコープ外（MVP時点）

- getter相当の「アクセサ関数経由」追跡（第3段階）
- `pycparser` によるC言語AST解析（正規表現で十分な精度を確保する）
- extern変数のヘッダ経由追跡（複雑すぎるため後回し）
- `.c` ファイル（Pro*Cではない純粋なC）への対応
- Javaツールとのプラグイン統合（3言語が揃った後にリファクタリング）
